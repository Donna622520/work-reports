## 🎯 关键经验总结

### 1. 大文档生成策略
**问题**: 直接输出完整 HTML 会被截断  
**解决**: 使用 `cat >> file.html` 分段写入，最后移动文件  
**适用场景**: 任何大内容生成任务

### 2. 飞书通知规范
**格式**: `[显示文本](URL)`  
**注意**: 纯文本 URL 会显示为黑色字符串  
**工具**: `send_message` 工具，target 格式 `feishu:chat_id`

### 3. GitHub 认证最佳实践
**避免**: `git remote set-url origin https://x-access-token:TOKEN@...`  
**推荐**: 使用 `gh` CLI 或标准 HTTPS URL  
**原因**: 空 token 会导致认证失败

### 4. 工作日报自动化
**定时**: 每日 22:00  
**手动**: 22:00 后可手动更新  
**优势**: 自动总结 + 手动补充，确保完整性
