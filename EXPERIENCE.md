# 工作日报经验沉淀 (Work Report Experience)

> 记录工作日报自动化系统的成功经验、失败教训和解决方案。
> 使用方法：遇到问题时先搜索此文件，避免重复犯错。

---

## 📝 项目概述

工作日报自动化系统是一个本地化的工作内容汇总工具，通过读取本地日志、终端历史、飞书消息等数据源，自动生成 Markdown 格式的工作日报，并通过飞书推送到用户。

**核心特点**：
- 不需要外部资源（只读取本地文件）
- 功能固定（汇总工作内容）
- 路径明确（`/home/donna/Hermes/reports/`）

## ✅ 成功经验

### 1. 交付路径一致性原则

**问题**：之前工作日报的生成路径、飞书发送路径、用户访问路径不一致，导致用户找不到报告。

**解决方案**：
- **生成路径** = `/home/donna/Hermes/reports/YYYYMM/YYYYMMDD_工作日报.md`
- **飞书发送路径** = 生成路径（直接发送）
- **用户访问路径** = 生成路径（用户可以直接访问）

**核心原则**：
1. **用户可见性优先**：不使用隐藏目录（如 `~/.hermes/docs/`）
2. **路径一致性**：生成路径 = 飞书发送路径 = 用户访问路径
3. **明确归档策略**：按月归档（`YYYYMM/` 目录）

**验证方法**：
- 报告生成后，立即检查路径是否正确
- 飞书发送后，确认用户可以直接访问
- 用户访问时，路径与生成路径一致

**参考技能**：`work-report-delivery-standards`

### 2. 文件三件套规范

**问题**：之前工作日报只有 README，缺少 SKILL 和 EXPERIENCE 文件，导致 AI 执行任务时缺乏明确指引，历史问题没有沉淀。

**解决方案**：
创建完整的三件套文档：

#### README.md（用户视角）
- **内容**：项目概述、功能特性、快速开始、配置说明、使用示例
- **用途**：用户了解项目是什么，如何使用
- **位置**：`/home/donna/Hermes/reports/README.md`

#### SKILL.md（AI 视角）
- **内容**：工作流总览、规范约束、执行步骤、验收标准
- **用途**：AI 执行任务时的操作指南
- **位置**：`/home/donna/Hermes/reports/SKILL.md`

#### EXPERIENCE.md（问题视角）
- **内容**：成功经验、失败教训、验证经验、在哪里错了就停止
- **用途**：记录历史问题，避免重复犯错
- **位置**：`/home/donna/Hermes/reports/EXPERIENCE.md`

**使用原则**：
1. **遇到问题先搜索**：先搜索 EXPERIENCE.md，看是否有历史记录
2. **执行任务看 SKILL**：AI 执行任务时严格按照 SKILL.md 的步骤
3. **用户引导看 README**：用户需要了解项目时，参考 README.md
4. **职责分离**：三件套互不重复，各司其职

## ❌ 失败教训

### 1. 资源库滥用

**问题**：误以为工作日报需要资源库，创建了不必要的文档。

**原因分析**：
- 没有理解资源库的真正用途
- 资源库 = 外部资源列表（新闻源、技能市场、工具推荐）
- 本地技能（README/SKILL/EXPERIENCE）≠ 资源库
- 工作日报不需要外部资源搜索，所以不需要资源库

**正确做法**：
- 工作日报只需要三件套（README + SKILL + EXPERIENCE）
- 不需要额外的资源库文档
- 本地技能已经固化在工作流中，不需要单独的资源库

**教训**：
- 在创建任何文档之前，先思考"这个文档是否真的需要？"
- 资源库只用于记录外部资源，不用于记录本地技能
- 本地技能应该固化为 `.hermes/skills/` 下的技能文件

---

### 2. 路径不一致

**问题**：工作日报的生成路径、飞书发送路径、用户访问路径不一致。

**原因分析**：
- 自动化脚本生成到 `/home/donna/Hermes/reports/`
- 手动复制到 `~/.hermes/docs/`（隐藏目录）
- 用户访问时找不到报告

**正确做法**：
- 统一使用 `/home/donna/Hermes/reports/YYYYMM/YYYYMMDD_工作日报.md`
- 飞书发送时直接发送这个路径
- 用户访问时直接使用这个路径
- 禁止使用隐藏目录（如 `~/.hermes/docs/`）
